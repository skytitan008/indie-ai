## 💬 常见问题 (FAQ)

### Q1: 为什么不使用大模型？

**A**: 大模型有几个固有问题：
1. **黑盒决策** - 无法解释为什么做这个决定
2. **成本高昂** - API 调用费用高，本地部署需要高端 GPU
3. **不可控** - 输出不稳定，难以精确控制
4. **隐私问题** - 数据需要发送到外部服务器

我们的项目探索另一条路：用强化学习实现透明、可控、低成本的 AI。

### Q2: 和大模型比，优势在哪？

**A**: 
- ✅ **透明度** - 每个决策都有明确理由
- ✅ **成本** - 普通电脑就能运行
- ✅ **可控** - 参数可调，行为可预测
- ✅ **隐私** - 完全本地运行
- ✅ **学习** - 持续从经验中学习进步

### Q3: 适用场景有哪些？

**A**: 
- 项目任务管理
- 日常工作辅助
- 学习规划
- 多 Agent 协作研究
- AI 教育演示

### Q4: 学习算法如何选择？

**A**: 
- **Q-Learning** - 适合探索性任务，Off-policy
- **SARSA** - 适合保守任务，On-policy

实验显示 Q-Learning 收敛更快，SARSA 更稳定。

### Q5: 多 Agent 如何协作？

**A**: 
基于消息传递的协作系统：
1. 协调器广播任务
2. Agent 根据技能投标
3. 协调器评分选择
4. 分配任务并跟踪

支持动态负载均衡和信任度机制。

**正文**:

### 背景

在大模型横行的今天，我们想探索另一条路：不依赖大模型，能否实现有用的 AI 系统？

### 项目目标

1. **透明可解释** - 每个决策都有明确理由
2. **轻量级** - 普通电脑就能运行
3. **可定制** - 灵活的配置系统
4. **实用导向** - 真正帮助完成实际工作

### 技术架构

采用符号 AI + 强化学习的混合架构：

- **决策引擎**: 基于效用函数的透明决策
- **学习系统**: Q-Learning + SARSA 双算法
- **记忆系统**: SQLite 持久化存储
- **执行系统**: 真实任务执行（代码格式化、测试、日报）
- **协作系统**: 多 Agent 消息传递

### 核心亮点

1. **透明决策**
   - 效用函数 = 紧急度×35% + 重要性×35% + 效率×20% + 依赖性×10%
   - 每个决策都可追溯、可解释

2. **经验学习**
   - Q-Learning: Off-policy，快速探索
   - SARSA: On-policy，稳定收敛
   - 100 轮实验验证学习曲线

3. **多 Agent 协作**
   - 基于技能匹配的任务分配
   - 动态负载均衡
   - 信任度机制

4. **真实任务**
   - 不是演示，是真的帮你干活
   - 代码格式化、测试运行、日报生成

5. **Web 可视化**
   - 学习曲线、Q 表增长、成功率图表
   - 实时数据、深色模式、响应式设计

### 实验数据

- **质量优先配置**: 98% 完成率
- **学习曲线**: Q 表 0→89，1888 次更新
- **长期学习**: 100 轮收敛验证
- **多 Agent**: 2 个 Agent 协作完成 6 个任务
